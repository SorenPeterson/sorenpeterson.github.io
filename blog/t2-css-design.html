Designing with CSS

Because it's basically the only option

11.9.14

Margin, border and padding make up what is know as the box model. The box model refers to the space in and around the element. At the very center there is the content. This can be anything from text and images to entire divs. The next layer is the padding. The padding is the amount of space around the content. Background colors will be applied to the padding. This is the main function of padding. When creating a button you could add more padding to make the button bigger but keep the text the same size. Wrapped around the padding is the border. The border is a line or border image that wraps around the element. Some common borders are solid, dashed and dotted lines. One could also use a specially formatted image to create a really fancy border around their element. The last and outermost layer is the margin. This is the space between the element and other elements. By setting a margin you can push the element away from others.

Classes and ID's are used to select specific html elements to apply styles to. A class and an id are very similar in that they both allow us to set special names to access our elements. The difference between the two is that while we can have as many elements of a certain class type as we want we can only have one element of a certain id type. Classes are good for commonly recurring elements such as buttons. Id's are good for major structural elements like headers or other unique elements like navbars. One important thing to note is that an element can have both a class and an id specified. It will inherit styling from both.

CSS positioning allows us to place our HTML elements exactly where we want to. Relative positiong places the element relative to where it would normally be placed. Use it with a :hover for a cool effect like the button is being pressed. Absolute positioning allows to place the element exactly based on the document. The element will still scroll though. Fixed positioning fixes the element to the browser window. When you scroll the element will not move.

Elements in html can display themselves in a few different ways. There are inline elements and block elements. Inline elements fall into the flow of content naturally. Their size is determined by their contents. It is as such that inline elements are usually the content. Block elements display on their own line. They will not flow with other inline elements. Blocks can be given a size and easily positioned. Blocks are thusly suited for containing content. There is also a type called inline-block. Inline blocks flow with other inline elements but they can retain their sizing property's.
